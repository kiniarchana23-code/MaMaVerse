MaMaVerse: Responsible, Medically Curated AI Assistance for Pregnancy & Parenthood

Deployed Architecture and Agentic Ecosystem for the "Agent for Good" Submission Track
Track: Agent for Good

Note for Reviewers: This submission presents a fully functional production platform deployed in Mumbai (asia-south1) on Google Cloud Run, integrating Google Gemini 2.5 models and Cloud Firestore. It is designed around safety-first guidelines (WHO, ICMR-NIN, and AAP) under a Human-in-the-Loop review flow.

1. Problem Definition & Vision

1.1 The Maternal Information Crisis

For pregnant women and new mothers, the transition into parenthood is accompanied by a massive influx of physical, psychological, and dietary changes. Sifting through search engines, parenting blogs, and social video platforms leads to three severe issues:

  • Anxiety and Overwhelm: Conflicting advice on minor symptoms causes unnecessary stress and anxiety during a delicate biological period.
  • Unverified & Dangerous Advice: Search algorithms prioritize engagement over medical validity, exposing mothers to unsafe home remedies, unscientific diets, or outdated sleep practices.
  • Lack of Personalization: A mother at week 12 of pregnancy requires completely different nutritional, test, and wellness recommendations compared to a mother at week 36 or a new mom with a 3-month-old infant.

1.2 Vision: The "Agent for Good" Grounding

To solve this, MaMaVerse establishes a Responsible AI Pipeline that ensures zero unverified content reaches the user:

  • Medical Sourcing: AI ingestion is restricted to reputable domains (WHO, ICMR-NIN, NHS, AAP, CDC).
  • Strict Admin Validation: AI-generated article drafts must be reviewed, edited, and approved by a human administrator in the Admin Portal before they are published to the database.
  • Phase-Specific Personalization: Specialized agents adapt weekly and monthly summaries, custom vegetarian/vegan diet plans, and wellness guidelines to the exact week of pregnancy or child's age.

2.1 Backend Pipeline (FastAPI)

The backend is written in Python using FastAPI for high performance and automatic OpenAPI schema generation. It uses Pydantic for rigid input validation and SlowAPI for client-specific rate limiting to protect API endpoints from abuse.

2.2 Frontend Client (Next.js 14)

The user interface is built on Next.js 14 utilizing React 18, vanilla CSS, and Framer Motion for premium, fluid layout transitions. It includes:

  • Authentication Hub: Seamless Google Sign-In and a frictionless Guest Mode.
  • Personalization Wizard: A responsive slider enabling users to choose their current phase (pregnancy week or baby age in months) and dietary preferences (Vegetarian, Non-Vegetarian, Vegan).
  • Dynamic Dashboard: Displays published, medically-sourced articles matching the user's phase.
  • AI Chat Interface: Interactive chat pane letting users query specific agents (Pregnancy, New Mom, Nutrition, Wellness, Healthcare, Admin).

3. Specialized Multi-Agent Ecosystem

The core value of MaMaVerse lies in its network of Specialized AI Agents, each given distinct system prompts, medical context parameters, and temperature configurations to excel at their respective domains:

  • Pregnancy Knowledge Agent: Guides expecting mothers through fetal development, trimester-specific symptoms, and prenatal tests. Metaphors are used to describe fetal growth (e.g. week 12 is a lemon).
  • New Mom Agent: Guides postpartum recovery, breastfeeding, vaccination schedules, and safe newborn sleep (preventing SIDS).
  • Nutrition Agent: Creates custom 7-day Indian meal plans grounded in ICMR-NIN guidelines.
  • Wellness Agent: Supports mental wellness with breathing exercises and postpartum crisis hotlines.
  • Healthcare Discovery Agent: Connects to the Google Places API to search local pediatricians/gynecologists by city name.
  • Admin Ingestion Agent: Ingests new medical URLs, analyzes duplicate risk, and prepares drafts for human review.

4. Implementation Quality, Security & Privacy

4.1 Security & RBAC

Firebase Auth is integrated with backend verification. Admin actions are secured with custom claims (`role: admin`). Guest profiles are kept strictly in browser `sessionStorage` for absolute user privacy.

4.2 In-Memory Database Sorting

To avoid complex Firestore composite indexes, Python performs document sorting and pagination in-memory. This guarantees high uptime, easy database maintenance, and fast response times.

4.3 Hydration Resilience

Firebase module initialization is safely wrapped in a try-catch interface. This, paired with a 3-second safety timer in AuthContext and a route guard in Axios response interceptor, completely prevents client-side rendering crashes and redirect loops.

5. Overall User Value & Project Story

MaMaVerse represents a secure, functional blueprint of how AI agents can support society. By combining rigorous Human-in-the-Loop review, localized nutrition guides, and a highly responsive UX, MaMaVerse empowers expecting parents with the safe information they need, when they need it most.
